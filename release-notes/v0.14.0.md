# Phantty v0.14.0

This release adds an embedded WebView2 browser panel so local development pages
and remote web services can be opened without leaving Phantty.

## Highlights

- Added a right-side WebView2 browser panel, available from the command center
  with `Ctrl+Shift+P` and `Toggle Browser`.
- Added Ctrl-click support for `http://` and `https://` URLs in terminal
  output, opening them in the embedded browser instead of an external browser.
- Added an in-panel URL bar: click it, type or paste an address, then press
  `Enter` to navigate.
- Added automatic scheme handling for typed addresses: localhost-style entries
  use `http://`, while public hostnames default to `https://`.
- Added drag resizing for the browser panel so it can sit beside the terminal,
  File Explorer, and Markdown preview panel.

## SSH and local development workflow

Phantty now understands the common "server is running on the remote host"
workflow. In SSH profile sessions, Ctrl-clicking loopback URLs such as
`http://127.0.0.1:4232` or `http://localhost:43455` opens them through an
automatic local SSH tunnel. Public URLs and private network URLs such as
`https://10.10.x.x` still open directly in the WebView2 panel.

The browser also normalizes local wildcard URLs. Opening `0.0.0.0:1234` is
mapped to a local browser-safe loopback URL, avoiding terminal resize escape
sequence fallout from handing that text to the terminal path.

## Packaging

GitHub releases now publish two portable Windows zip archives:

- `phantty-windows-portable-v0.14.0.zip` for the lightweight terminal package.
- `phantty-windows-portable-webview2-v0.14.0.zip` with `WebView2Loader.dll` for
  the embedded browser panel.

The WebView2 package expects the Microsoft Edge WebView2 Runtime to be
available on Windows, which is normally already installed on current Windows
systems.

## Fixes and polish

- Fixed WebView2 child-window lifetime and focus handling when the panel is
  closed, reopened, or resized.
- Reused active SSH browser tunnels when possible, instead of spawning a new
  tunnel for the same remote endpoint.
- Kept OpenSSH helper stderr visible so tunnel failures retain the underlying
  SSH diagnostic.
- Added release packaging checks for both portable variants.
